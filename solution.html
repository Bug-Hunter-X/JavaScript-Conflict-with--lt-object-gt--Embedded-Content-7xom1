To solve the issue of JavaScript conflicts with embedded content in HTML using the <object> tag, a good approach is to use Namespaces. This helps avoid naming collisions and ensures that each JavaScript code runs in its own isolated context, preventing unexpected interference.

In the example below, we wrap our scripts in an Immediately Invoked Function Expression (IIFE) to create a private scope.

<!DOCTYPE html>
<html>
<head>
<title>Solution</title>
<script>
  (function() {
    // Namespace for main page scripts
    window.myApp = {};
    myApp.myVariable = "Value from main page";
    myApp.myFunction = function() {
      console.log("Main page function");
    }
  })();
</script>
</head>
<body>
  <object data="mydocument.pdf" type="application/pdf">
    <param name="scroll" value="yes">
  </object>
<script>
  //Example of how to access the variable from the IIFE in the main app context
  console.log(myApp.myVariable);
  myApp.myFunction();
</script>
</body>
</html>

Remember to replace "mydocument.pdf" with the actual path to your PDF file. This approach helps to isolate the JavaScript within the embedded object from the main page's JavaScript, avoiding potential conflicts.
